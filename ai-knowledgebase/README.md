# AI Knowledge Base Guidelines

This folder contains static markdown files that serve as the single source of truth for the AI:

- product_overview.md — describes what manu2print does
- features.md — lists only real features
- workflows.md — explains how users complete tasks
- troubleshooting.md — factual guidance for common support issues
- limitations.md — known product limitations
- faq.md — common user questions and answers
- competitive-context.md — market landscape and competitor context for KDP + Kindle (competitive-positioning queries only; update from reliable sources)
- ai-instructions.md — instructions for the AI voice (persona, knowledge-base-only answers, templates, competitive-positioning format)

## Update Checklist

Whenever adding or changing tools or workflows:

1. Update product_overview.md if the high-level description changes.
2. Update features.md to reflect real features only.
3. Update workflows.md if user tasks change.
4. Update troubleshooting.md with new support guidance.
5. Update limitations.md if product capabilities change.
6. Update faq.md with relevant Q&A.
7. Update competitive-context.md when you have reliable competitor or market information (do not speculate).
8. Keep all content factual and concise.
9. Do not connect any files to code or the system unless intentionally implementing a new automation.

**Notes:**
- This folder is static reference material.
- The AI will use it as a source of truth.
- Always maintain accuracy to prevent hallucinations.
